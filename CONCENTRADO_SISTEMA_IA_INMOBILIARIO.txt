CONCENTRADO SISTEMA IA INMOBILIARIO
===================================

1. VISIÓN GENERAL
-----------------

Este proyecto es una plataforma de **análisis de mercado inmobiliario con inteligencia artificial**, diseñada para identificar las mejores oportunidades de inversión en terrenos y propiedades en México.

Combina en un solo sistema:
- Un **mapa interactivo** que muestra zonas con mayor o menor potencial de plusvalía.
- Un **modelo de Machine Learning** que estima precios y potencial de crecimiento.
- Una **base de datos inmobiliaria y geoespacial** con datos reales.
- Un **chatbot inteligente** que actúa como asesor inmobiliario especializado.

Está pensada para:
- Desarrolladores inmobiliarios.
- Inversionistas y fondos.
- Despachos de consultoría.
- Asesores y brokers inmobiliarios.

Objetivo principal: **reducir la incertidumbre** y **acelerar la toma de decisiones** usando datos cuantitativos, visualizaciones claras y explicaciones en lenguaje simple.


2. PROBLEMA QUE RESUELVE
------------------------

En el mercado inmobiliario tradicional:
- La información está dispersa en muchos portales.
- Los datos son poco estructurados o poco confiables.
- Las decisiones suelen basarse en intuición, no en evidencia cuantitativa.

Este sistema ayuda a responder preguntas clave como:
- ¿En qué zona me conviene invertir?
- ¿Este terreno está caro o barato contra el mercado?
- ¿Qué plusvalía puedo esperar en esta zona a mediano plazo?

En lugar de depender solo de percepciones humanas, **integra datos, modelos estadísticos y visualizaciones** para ofrecer una evaluación objetiva de cada zona.


3. COMPONENTES DEL SISTEMA
--------------------------

El sistema se compone de cuatro grandes módulos:
1) Mapa interactivo de oportunidades.
2) Motor de Machine Learning (IA).
3) Base de datos inmobiliaria y geoespacial.
4) Chatbot asesor inmobiliario (Favier AI).


3.1 Mapa interactivo de oportunidades
-------------------------------------

Es la “cara” visible del sistema para el usuario final:
- Un **heatmap** (mapa de calor) pinta las zonas según su nivel de plusvalía y atractivo.
- Puntos o celdas representan áreas analizadas; al hacer clic se muestra:
  - Precio estimado por metro cuadrado.
  - Score de plusvalía (0 a 100).
  - Potencial de crecimiento (bajo, medio, alto, muy alto).
  - Nivel de riesgo y nivel de confianza.

El usuario puede aplicar filtros:
- Ciudad y estado.
- Rango de precios.
- Score mínimo de plusvalía.
- Otros criterios según el perfil del inversionista.

Beneficio para el cliente:
- Ver en segundos **dónde están las mejores zonas para invertir**.
- Identificar áreas “calientes” (alto potencial) y áreas de riesgo.


3.2 Motor de Machine Learning (IA)
----------------------------------

Detrás del mapa hay un modelo de Machine Learning que realiza los cálculos de precio y plusvalía.

- Algoritmo principal: **Random Forest Regressor** (modelo supervisado).
- Entrenado con **miles de registros de propiedades y datos de contexto**.

Variables (features) usadas, entre otras:
- Ubicación geográfica (latitud, longitud, ciudad, estado).
- Tamaño del terreno (m²).
- Distancia al centro de la ciudad.
- Densidad poblacional y nivel socioeconómico (INEGI).
- Amenidades cercanas (escuelas, hospitales, comercios, etc.).
- Información temporal e histórica de precios.

El modelo produce:
- Precio estimado por metro cuadrado.
- Score de plusvalía (0–100).
- Categoría de potencial (bajo, medio, alto, muy alto).
- Nivel de confianza de la predicción.


3.3 Base de datos inmobiliaria y geoespacial
--------------------------------------------

La base de datos (PostgreSQL/Supabase) almacena toda la información que alimenta al modelo y al mapa:
- Propiedades comparables (precios, superficies, direcciones, coordenadas).
- Amenidades (escuelas, hospitales, comercios…) desde OpenStreetMap.
- Datos demográficos y socioeconómicos del INEGI.
- Predicciones del modelo (precio m², score de plusvalía, potencial, riesgo, etc.).
- Históricos de precios por ciudad y zona.

Permite:
- Consultar de forma rápida grandes volúmenes de información.
- Revisar el historial y la evolución de precios por zona.
- Retroalimentar el modelo con datos nuevos a medida que el mercado cambia.


3.4 Chatbot asesor inmobiliario (Favier AI)
-------------------------------------------

Además del mapa, el sistema incluye un **chatbot inteligente**:
- Construido con **n8n + OpenAI**.
- Conectado directamente a la misma base de datos de predicciones y tablas de mercado.

Puede responder, por ejemplo:
- “¿Cuáles son las mejores zonas en Guadalajara para invertir a 5 años?”
- “Compárame dos zonas específicas en Querétaro.”
- “Explícame el nivel de riesgo de esta zona y por qué.”
- “Dame las 5 mejores oportunidades en tal ciudad con presupuesto de X.”

Claves:
- El chatbot **no inventa datos**: consulta Supabase y los resultados del modelo ML.
- Responde en **lenguaje natural**, entendible incluso para clientes no técnicos.


4. FLUJO DE FUNCIONAMIENTO
--------------------------

El funcionamiento completo puede resumirse así:

1) **Datos → Modelo**
   - Se obtienen datos de propiedades, amenidades, INEGI e históricos.
   - Se limpian y transforman.
   - El modelo de Machine Learning se entrena y se actualiza periódicamente.

2) **Modelo → Base de datos**
   - El modelo genera miles de predicciones por ciudad, estado y zona.
   - Cada predicción (precio, score, potencial, riesgo, confianza) se guarda en Supabase.

3) **Base de datos → Mapa y chatbot**
   - El frontend consulta la API y pinta el mapa interactivo.
   - El chatbot usa los mismos datos para responder preguntas y justificar recomendaciones.

4) **Usuario final**
   - Explora el mapa, aplica filtros y compara zonas.
   - Consulta al chatbot para aclarar dudas.
   - Toma decisiones de inversión basadas en datos y explicaciones claras.


5. TECNOLOGÍAS UTILIZADAS
-------------------------

Frontend:
- **Angular**: framework principal de la aplicación web.
- **Leaflet**: mapas interactivos, puntos, clusters y heatmaps.
- **HTML, CSS, TypeScript**: tecnologías base.

Backend y ML:
- **Python + FastAPI**: API REST para predicciones y datos (`/predictions/heatmap`, `/predictions/nearby`, `/predict`, etc.).
- **scikit-learn**: implementación del modelo Random Forest.
- **Pandas / NumPy**: limpieza y preparación de datos.

Datos:
- **PostgreSQL + Supabase**: base de datos central del sistema.
- **OpenStreetMap (OSM)**: amenidades reales por zona.
- **INEGI**: datos demográficos y socioeconómicos.

IA conversacional y automatización:
- **n8n**: orquestación de workflows (webhooks, consultas a BD, llamada al LLM).
- **OpenAI (LLM)**: modelo de lenguaje para generar respuestas en texto.


6. ENTRENAMIENTO DEL MODELO DE ML
---------------------------------

1) **Recolección de datos**
   - Propiedades: precio, superficie (m²), ubicación, ciudad y estado.
   - Contexto: amenidades cercanas, datos del INEGI, históricos de precios.

2) **Preparación y creación de variables (feature engineering)**
   - Limpieza: eliminación de valores nulos, duplicados y outliers extremos.
   - Cálculo de:
     - `price_m2 = precio_total / área_m2`.
     - `distance_to_center` (distancia al centro de la ciudad).
     - `population_density`, `economic_level_score` (desde INEGI).
   - Conversión de categorías (ciudad, estado, etc.) a números y normalización de escalas cuando hace falta.

3) **Entrenamiento**
   - Algoritmo: **Random Forest Regressor**.
   - División de datos:
     - Entrenamiento: ~80%.
     - Prueba: ~20%.
   - Métricas de evaluación:
     - **R²**: porcentaje de la variación en el precio que el modelo explica.
     - **MAE**: error promedio en pesos por metro cuadrado.
   - Cuando las métricas son buenas (por ejemplo R² ≈ 0.75–0.85 y MAE razonable), se guarda el modelo entrenado en un archivo `.pkl` junto con las transformaciones necesarias.

4) **Despliegue**
   - El backend carga el modelo entrenado al iniciar el servidor.
   - Cada vez que llega una petición de predicción, aplica las mismas transformaciones y llama al modelo para obtener una respuesta en tiempo real.


7. CÓMO FUNCIONAN LAS PREDICCIONES
----------------------------------

Flujo técnico simplificado:

1) El frontend envía una petición a la API (por ejemplo a `/predict`):
   - Coordenadas (latitud y longitud).
   - Área del terreno en m².
   - Ciudad y estado.

2) El backend:
   - Transforma estos datos igual que en el entrenamiento (cálculo de `distance_to_center`, codificación de ciudad/estado, etc.).
   - Construye un vector de características y lo pasa al modelo.

3) El modelo:
   - Cada árbol del Random Forest estima un **precio por m²**.
   - Se promedian todas las salidas para obtener un **precio_m2 estimado único**.
   - A partir de ese precio y referencias de mercado:
     - Se calcula un **score de plusvalía** (0–100).
     - Se traduce en **potencial de crecimiento** (bajo/medio/alto/muy alto).
     - Se estima un **nivel de confianza** según el desempeño histórico.

4) Respuesta:
   - El backend devuelve un JSON con:
     - `predicted_price_m2`
     - `plusvalia_score`
     - `growth_potential`
     - `risk_level` (si aplica)
     - `confidence`
   - El frontend usa esto para:
     - Pintar el heatmap.
     - Mostrar popups en el mapa.
     - Alimentar respuestas del chatbot.


8. INTERPRETACIÓN DE LAS MÉTRICAS
---------------------------------

- **Precio por metro cuadrado (`precio_m2`)**
  - Es el **precio estimado de venta por cada m²** de terreno o propiedad.
  - Ejemplo: si un terreno de 500 m² tiene un precio estimado de 10,000,000 MXN, entonces `precio_m2 ≈ 20,000 MXN/m²`.
  - Permite comparar propiedades de distinto tamaño de forma justa.

- **Score de plusvalía (0–100)**
  - Indica **qué tan atractiva es la zona** en términos de potencial de incremento de valor.
  - Se calcula combinando:
    - El precio estimado frente a promedios históricos.
    - Datos de contexto (demografía, amenidades, etc.).
  - Interpretación típica:
    - 0–39: bajo potencial.
    - 40–59: potencial medio.
    - 60–79: buen potencial.
    - 80–100: potencial muy alto (oportunidades muy interesantes).

- **Potencial de crecimiento**
  - Traduce el score de plusvalía a una palabra:
    - “bajo”, “medio”, “alto” o “muy alto”.
  - Es la forma más simple de comunicar si una zona parece prometedora.

- **Nivel de riesgo**
  - Mide cuánta incertidumbre hay alrededor de la estimación.
  - Depende de:
    - Cantidad de datos en esa zona.
    - Variabilidad de precios.
    - Calidad y actualidad de la información.
  - Ejemplo: “buen potencial, pero riesgo medio” vs. “alto potencial y riesgo bajo”.

- **Nivel de confianza**
  - Es un **porcentaje aproximado** de qué tan seguro está el modelo de su predicción, basado en cómo ha funcionado antes con datos reales.
  - Por ejemplo, 85% de confianza indica que en situaciones similares el modelo suele acertar con buena precisión.

Forma sencilla de explicarlo a clientes:
- “Precio por m²”: cuánto debería valer, en promedio, cada m² comparado con el mercado.
- “Score de plusvalía”: una calificación del 0 al 100 que indica qué tan buena es la zona para invertir.
- “Potencial de crecimiento”: resumen del score en una palabra.
- “Riesgo”: cuánta incertidumbre hay alrededor de la estimación.
- “Confianza”: qué tan seguro está el sistema de lo que está diciendo.


9. SEGURIDAD Y CONFIABILIDAD
----------------------------

El sistema está diseñado para operar de forma segura en entornos reales:
- Todas las comunicaciones entre módulos usan **HTTPS**.
- El frontend solo tiene permisos de **lectura**; las escrituras se realizan desde servicios seguros.
- Las credenciales y claves sensibles se guardan como **variables de entorno**, no en el código.
- La base de datos utiliza:
  - Políticas de seguridad a nivel de fila (RLS).
  - Control de acceso por roles.
- Existen mecanismos de:
  - Limitación de peticiones (rate limiting).
  - Separación de entornos (desarrollo vs producción).

Resultado: **solución profesional**, no un prototipo, preparada para clientes reales y datos reales.


10. CASOS DE USO TÍPICOS
------------------------

1) **Inversionista que quiere diversificar**
   - Selecciona una ciudad.
   - Filtra por presupuesto y score de plusvalía.
   - El sistema muestra las mejores zonas y el chatbot resume por qué.

2) **Desarrollador que evalúa un terreno específico**
   - Hace clic en la ubicación exacta del terreno o introduce coordenadas.
   - Ve el precio estimado por m² y el potencial de plusvalía.
   - Compara esa zona con otras alternativas en la misma ciudad.

3) **Dirección o comité de inversión**
   - Revisa un dashboard con las principales ciudades y sus tendencias.
   - Usa el chatbot para generar explicaciones y argumentos para decisiones internas.
